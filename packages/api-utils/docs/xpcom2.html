Add-on SDK

Version 1.5

xpcom2

Using this module you can:

  • register a component with XPCOM, making it available to all XPCOM clients
  • retrieve a factory for a given XPCOM component
  • generate a UUID

The module also exposes the XPCOMUtils module.

API Reference

Classes

Factory

When a component is made available through XPCOM using the register() function, register() returns a Factory object that can be used to instantiate the component:

var myFactory = require("xpcom").register({name: "My Component",
                       contractID: "@me.org/myComponent",
                       create: MyComponent});

var myComponent = myFactory.createInstance(null, Ci.nsIMyComponent);

Factory wraps two key functions from nsIFactory: createInstance() and QueryInterface().

It also implements its own unregister() function, which removes the factory from XPCOM's registry, making the associated component unavailable.

Methods
createInstance(outer, iid)

Creates an instance of the component associated with this factory.

outer : nsISupports

This argument must be null, or the function throws NS_ERROR_NO_AGGREGATION.

iid : string

The interface ID of the interface being requested in the component to be created.

QueryInterface(interfaces)

Returns this, if the interfaces argument contains the interface ID for nsIFactory. Otherwise returns null.

interfaces : array

An array of interface IDs. If the array contains the interface ID for nsIFactory, then this is returned.

Returns: Factory
unregister()

Removes the factory from XPCOM's registry.

Functions

register(options)

Makes a component available through XPCOM.

Given a constructor for a component and some XPCOM metadata - a class ID, a contract ID and a name - this function creates and registers a factory for the component:

var xpcom = require("xpcom");

function HelloWorld() {}

HelloWorld.prototype = {
  QueryInterface: xpcom.utils.generateQI([Ci.nsIHelloWorld]),
  hello: function() {
    return "Hello World!";
  }
};

xpcom.register({name: "Hello World Component",
                contractID: "@mozilla.org/addonSDK/helloworld",
                create: HelloWorld});

XPCOM clients can subsequently access this factory and use it to create instances of the component.

var factory = require("xpcom").getClass("@mozilla.org/addonSDK/helloworld",
                                        Ci.nsIFactory);
var helloWorld = factory.createInstance(null, Ci.nsIHelloWorld);
helloWorld.hello();

register() returns a Factory object for the class, which wraps the nsIFactory QueryInterface() and createInstance() functions, as well as defining an unregister() function to remove the component from XPCOM.

options : object
[ uuid : nsIDPtr ]

A uuid which will be used as the class ID for this component. If you don't include this option, the register() function will generate a new uuid.

create : function

The constructor for the component.

name : string

A human-readable name for the component.

contractID : string

A human-readable string which will be used as the contract ID for the component. An XPCOM client will be able to use this value to access the component.

Returns: Factory

See the documentation for the Factory class in this page.

getClass(contractID, iid)

Returns the factory object for the class specified by contractID.

For example, given a registered XPCOM component which:

  • is identified with the contract ID "@me.org/myComponent"
  • implements the interface nsIMyInterface

we can access a factory and then use it to instantiate the component:

var myFactory = xpcom.getClass("@me.org/myComponent", Ci.nsIFactory);
var myComponent = myFactory.createInstance(null, Ci.nsIMyInterface);
contractID : string

The contract ID for the component whose factory will be returned.

[ iid : string ]

The interface type to be returned. If you don't include this option nsISupports is returned.

Returns: Object

The factory object. The type of this will depend on the value of the iid argument. If no iid argument is specified it will be of type nsISupports.

makeUuid()

Generates and returns a new uuid.

Returns: nsIDPtr

Properties

utils : object

The XPCOMUtils module.
